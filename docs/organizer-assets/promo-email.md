## Body

Hi all,

We’re launching **Permission to Run**, an Agent Experience (AX) hackathon for Drupal.

The idea is simple: build with **any agent** — Drupal’s built-in agent, Cursor, Claude Code, Codex, your own tooling, or a team of agents — while Drupal stays “the governor” through permissions, workflows, schema validation, diffs, and auditability.

### What counts
To qualify, your submission needs:
- a real “agent did useful work” moment
- at least one AX artifact, such as `AGENTS.md`, a skill/runbook, a validator, a tool mapping, or a benchmark task
- an Agent Experience Report describing what worked, what was confusing, and what would make the next run faster
- a named team lead who is physically attending DrupalCon Chicago 2026 if you want the entry to be prize-eligible

### Why join
This is about more than demos. We want practical patterns that make agent work in Drupal safer, more open, and more repeatable.

### Examples
- draft content safely with revisions and moderation
- propose config changes as reviewable diffs
- build safe operator flows for site ops
- improve the guidance, validation, and interfaces agents need to succeed
