# Skill: write the Agent Experience Report

## Goal

Turn the build session into reusable learning for the next principal.

## Prompt pattern

Use this prompt with your agent after the build:

> Review the work completed in this repository and produce an Agent Experience Report for the Drupal Permission to Run hackathon. Summarize:
> 1. What worked well
> 2. What was confusing or ambiguous
> 3. What you could not do, and why
> 4. What guidance, validation, or interface would make the next run 10× faster
> 5. Which parts of Drupal acted as useful guardrails
> Keep it concrete and evidence-based.

## What good looks like

A strong report is:
- specific
- honest
- actionable
- tied to real blockers
- helpful for future agents and maintainers

## Avoid

- generic praise
- vague complaints
- unsupported claims
- pretending a missing capability is a bug without evidence

## Bonus move

Turn the final report into:
- a Drupal issue
- a GitHub issue
- a follow-up benchmark task
