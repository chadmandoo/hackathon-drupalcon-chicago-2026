# Agent Experience Report

## What worked

Having a clear safety target — draft content, do not publish — made the workflow easy to execute.

## What was confusing

The exact field names and moderation states should be documented more explicitly.

## What the agent could not do

The agent could not confirm site-specific taxonomy IDs because no live site context was provided.

## What would make the next run 10× faster

A field map, example payload, and validation script would remove most guesswork.

## Where Drupal acted as a useful governor

Schema requirements and moderation workflow kept the output reviewable and safe.

## Recommended follow-up issue(s)

- Add a sample content payload to the starter kit
- Document common moderation states
