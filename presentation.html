01 — The Vision

Drupal + Desktop AI = The Editor's New Best Friend

What if your editorial team could manage a Drupal site from their desktop AI assistant — the same way developers use Cursor for code?

The idea: Connect Claude Desktop (or any MCP-capable AI) to your Drupal site. The AI becomes the editorial interface. Drupal stays the governor.

Editors work in their own environment — local files, research, notes, brand guidelines — and the AI bridges the gap to the CMS. No need to build complex editorial UIs on the web. The AI wraps the experience.

Why this is better than embedded AI

Embedded AI (in Drupal)Desktop AI (via MCP)

02 — What We Built

The Prompt Post — A Living Demo

A satirical AI news publication running on Drupal 11:

  • 32 MCP tools — 12 custom editorial tools + 20 from contrib
  • 3-tier permission model — Writer (22 tools) → Reviewer (23) → Admin (32)
  • Editorial workflow — draft → review → published → archived
  • Decoupled React SPA — newspaper-style frontend
  • OAuth 2.1 + PKCE + Dynamic Client Registration — paste a URL, log in, done
  • 15+ satirical articles, 10 job listings, Sudoku puzzle generator

The demo: Claude creates articles, submits for review, publishes (if authorized), flags breaking news, generates puzzles. A Writer literally cannot publish. Drupal enforces it. Changes appear on the live site instantly.

03 — Drupal as Governor

Three Layers of Governance

Layer 1: Discovery-time filtering

Our hook_mcp_server_enabled_tools_alter() removes tools the user can't access before the agent sees them. A Writer never knows user_block exists.

Layer 2: Execution-time permission checks

Every tool checks Drupal permissions. The content_publisher returns: "Access denied. Required permission: use editorial transition publish. Your roles: editor."

Layer 3: Content moderation workflow

draft → review → published → archived. Content must pass through review. Same workflow for humans and AI.

05 — Visual Content Planning

See It Before You Ship It

With the decoupled SPA, editors use Claude to:

  1. Draft content locally — Claude writes the article in conversation
  2. Preview visually — Claude can generate static HTML previews
  3. Iterate with feedback — "Shorter headline, add a pull quote, change category"
  4. Publish when ready — One command sends it to Drupal via MCP

Content never touches Drupal until the editor is satisfied. No draft pollution. No incomplete nodes. The AI handles the messy creative process; Drupal gets the clean result.

07 — Challenges & Lessons

What We Ran Into

MCP Server Module — Works, but setup is hard

Requires understanding OAuth 2.1, PKCE, and Dynamic Client Registration simultaneously. We patched the module for tool filtering, server instructions, and a JSON Schema bug. Developer-first today — needs a setup wizard.

Claude Desktop OAuth — Caching friction

Switching users requires full disconnect/reconnect. Tool list persists after permission changes. UX friction, not deal-breakers.

MCP client support — Growing but uneven

Claude: full support, works great. ChatGPT: web yes, desktop no. The spec is open — adoption will follow.

Per-tool authorization

Users must grant access to each tool on first connect. This is a feature: explicit human consent on top of Drupal's permissions.

What AI didn't do well

Honestly — not much. The bottleneck was setup complexity and module maturity, not AI capability. Claude built 4 modules, 12 tools, 15 articles, and all documentation in one session.

09 — Contribution Back to Drupal

What We're Giving Back

Contrib patch: hook_mcp_server_enabled_tools_alter()

A standard Drupal alter hook that lets any module filter the MCP tools list at discovery time. Our implementation filters by user permissions — but the hook is generic. Any site can use it for role-based, scope-based, or context-based tool filtering.

Also in the patch

  • Server instructions from config — auto-orients agents on connection
  • Fix for empty properties JSON serialization — broke Claude.ai's connector
  • DELETE method support — MCP spec compliance for session cleanup
